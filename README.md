# 🛠️ Princípios e Padrões de Projeto - Atividades Práticas

Este repositório foi criado para centralizar e organizar todas as atividades práticas, laboratórios e desafios desenvolvidos em sala de aula para a disciplina de **Princípios e Padrões de Projeto**. 

O objetivo principal é aplicar conceitos de engenharia de software, garantir a criação de códigos reutilizáveis, manuteníveis e flexíveis, além de dominar as melhores práticas de desenvolvimento orientado a objetos.

---

## 🎯 Objetivos de Aprendizado

*   **Princípios SOLID:** Aplicar os 5 princípios da POO para evitar rigidez, fragilidade e imobilidade no código.
*   **Padrões GoF (Gang of Four):** Implementar padrões Criacionais, Estruturais e Comportamentais.
*   **Refatoração:** Identificar *code smells* e aplicar padrões para melhorar o design do código sem alterar seu comportamento.

---

## 📂 Estrutura do Repositório

O projeto está organizado por tópicos e padrões estudados. Cada pasta contém a implementação do problema proposto e, quando necessário, uma breve explicação da solução adotada.

```text
├── 📌 principios-solid/         # Exercícios focados em SRP, OCP, LSP, ISP e DIP
├── 🏗️ padroes-criacionais/      # Exemplos de Factory Method, Abstract Factory, Singleton, Builder, etc.
├── 📐 padroes-estruturais/      # Exemplos de Adapter, Composite, Decorator, Facade, Proxy, etc.
└── 🔄 padroes-comportamentais/  # Exemplos de Strategy, Observer, State, Command, Template Method, etc.
